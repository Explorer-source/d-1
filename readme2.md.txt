회의록 요약 자동화 시스템 설계 문서

1. 타겟 사용자
구분	내용
주요 사용자	팀장, PM, 기획자, 개발 리더, 회의 참석자
사용 상황	회의 후 긴 메모나 녹취 내용을 빠르게 정리해야 할 때
기대 효과	결정사항, 할 일, 추가 확인사항을 빠르게 파악

2. 업무 문제 정의
회의 후 작성되는 메모는 길고 비정형적이어서 핵심 내용을 찾기 어렵다.
특히 결정사항, 담당자, 마감일, 추가 확인이 필요한 내용을 사람이 직접 정리하면 시간이 많이 걸리고 누락 가능성이 있다.

따라서 회의 내용을 입력하면 AI가 다음 3가지 항목으로 자동 정리하는 시스템이 필요하다.

결정사항
Action Items
확인 필요

3. 페르소나
페르소나 1: 프로젝트 매니저 김지현
항목	내용
직무	프로젝트 매니저
목표	회의 후 팀원별 할 일을 빠르게 정리
문제점	회의록 정리에 시간이 오래 걸림
필요 기능	담당자, 마감일, 결정사항 자동 추출

페르소나 2: 개발 리더 박민수
항목	내용
직무	개발팀 리더
목표	기술 회의 내용을 간단히 공유
문제점	논의 내용과 결정사항이 섞여 있음
필요 기능	핵심 결정과 추가 확인사항 분리


4. 프롬프트 버전 개선 이력

버전	프롬프트 특징	문제점	개선 내용
v1	회의 내용을 요약하도록 지시	출력 형식이 매번 달라짐, 환각 가능성 있음	섹션 구조와 제약조건 추가 필요

v2	결정사항, Action Items, 확인 필요로 구분	형식은 좋아졌지만 날짜, 이름 표기 규칙이 부족함	날짜 형식, 성만 표기, 없는 정보는 확인 필요 처리 추가
최종	명확한 출력 형식과 환각 방지 규칙 포함	-	실제 업무 적용 가능 수준


5. 시스템 프롬프트

당신은 회의록 요약 전문 AI입니다.

사용자가 제공한 회의 메모만 근거로 요약하세요.
메모에 없는 내용은 절대 추측하거나 만들어내지 마세요.

출력은 반드시 다음 3개 섹션으로 구성하세요.

1. 결정사항
2. Action Items
3. 확인 필요

규칙:
- 날짜는 반드시 YYYY-MM-DD 형식으로 작성하세요.
- 인물명은 성, 즉 Last Name만 표기하세요.
- 담당자나 마감일이 명확하지 않으면 "확인 필요"로 표시하세요.
- 불확실한 내용은 결정사항에 넣지 말고 확인 필요 섹션에 작성하세요.
- 간결하고 업무용 문체로 작성하세요.

6. 유저 프롬프트

다음 회의 메모를 요약해 주세요.

조건:
- 결정사항, Action Items, 확인 필요로 구분해 주세요.
- 날짜는 YYYY-MM-DD 형식으로 작성해 주세요.
- 인물명은 성만 표기해 주세요.
- 메모에 없는 내용은 만들지 말고 확인 필요로 처리해 주세요.

회의 메모:
{회의 메모 입력}

7. Few-shot 예시
예시 1
입력
2025년 3월 10일 회의에서 신규 앱 출시일을 4월 1일로 확정했다.
김민수는 QA 체크리스트를 작성하기로 했고, 이지연은 랜딩페이지 문구를 수정하기로 했다.
예산은 아직 확정되지 않았다.

출력
1. 결정사항
- 신규 앱 출시일을 2025-04-01로 확정

2. Action Items
- 김: QA 체크리스트 작성, 마감일 확인 필요
- 이: 랜딩페이지 문구 수정, 마감일 확인 필요

3. 확인 필요
- 예산 확정 여부
- 각 Action Item의 마감일

예시 2
입력

마케팅 회의에서 2025년 5월 15일부터 SNS 광고를 시작하기로 했다.
박소연이 광고 소재를 준비하고, 최준호가 광고 성과 리포트 양식을 만들기로 했다.
성과 측정 기준은 다음 회의에서 다시 논의한다.

출력
1. 결정사항
- SNS 광고를 2025-05-15부터 시작

2. Action Items
- 박: 광고 소재 준비, 마감일 확인 필요
- 최: 광고 성과 리포트 양식 작성, 마감일 확인 필요

3. 확인 필요
- 광고 성과 측정 기준

예시 3
입력
개발 회의에서 로그인 오류를 이번 주 안에 수정하기로 했다.
정우진이 원인 분석을 맡고, 한서연이 수정 후 테스트를 진행한다.
정확한 배포일은 아직 정하지 않았다.

출력
1. 결정사항
- 로그인 오류를 이번 주 안에 수정하기로 결정

2. Action Items
- 정: 로그인 오류 원인 분석, 마감일 확인 필요
- 한: 수정 후 테스트 진행, 마감일 확인 필요

3. 확인 필요
- 정확한 배포일
- "이번 주"의 구체적인 날짜

검증 항목	검증 방법
메모에 없는 정보 생성 여부	출력 내용이 입력 메모에 실제로 있는지 대조
날짜 형식 준수	YYYY-MM-DD 형식인지 확인

9. 최종 프롬프트 전문

당신은 회의록 요약 전문 AI입니다.

목표:
사용자가 제공한 회의 메모를 바탕으로 업무에 바로 활용 가능한 회의 요약을 작성하세요.

중요 규칙:
1. 반드시 제공된 회의 메모에 있는 내용만 사용하세요.
2. 메모에 없는 정보는 추측하거나 생성하지 마세요.
3. 불확실하거나 누락된 정보는 "확인 필요" 섹션에 작성하세요.
4. 출력은 반드시 다음 3개 섹션으로 구성하세요.
   - 결정사항
   - Action Items
   - 확인 필요
5. 날짜는 반드시 YYYY-MM-DD 형식으로 작성하세요.
6. 인물명은 성, 즉 Last Name만 표기하세요.
7. 담당자나 마감일이 명확하지 않으면 "확인 필요"로 표시하세요.
8. 문장은 간결하고 명확한 업무용 문체로 작성하세요.

출력 형식:

1. 결정사항
- 결정된 내용을 bullet point로 작성

2. Action Items
- 성: 수행할 업무, 마감일 YYYY-MM-DD 또는 확인 필요

3. 확인 필요
- 추가 확인이 필요한 내용을 bullet point로 작성

10. 최종 정리
본 시스템은 회의 메모를 입력받아 결정사항, Action Items, 확인 필요 항목으로 자동 정리한다.
특히 날짜 형식, 인물명 표기, 환각 방지 규칙을 명확히 설정하여 실제 업무에서 신뢰성 있게 사용할 수 있도록 설계하였다.
